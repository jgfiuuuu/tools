# 文献妙妙屋 · UI 重构方案（YoRHa-Inspired）

> 本文档是基于 YoRHa 设计语言的**重构级前端设计方案**，而非现有界面的优化迭代。
> 它面向"负责落地的模型或工程师"，描述全新的视觉系统、布局范式与交互语言。
> 约束边界：保留产品定位（文献工作台）、保留 API 契约、保留核心数据流，但 UI 层可大胆重构。

---

## A. 设计立场与产品理解

### 产品本质（不变）
这是一个**文献筛选与研究工作台**，不是聊天机器人、不是搜索引擎结果页、不是文档编辑器。核心动作是"输入研究问题 → 多源检索与筛选 → 逐篇精读判断 → 生成研究备忘录 → 本地留存复盘"。

### 当前问题（需破局）
现有 UI 走的是"暖米色纸感学术风"，安全但**缺少记忆点与辨识度**。三栏布局合理，但视觉语言过于平淡，像一个普通的 SaaS 工作台，没有让用户记住的"装置感"与"仪式感"。

### 重构目标
借鉴 YoRHa（《尼尔：机械纪元》UI 系统）的**工业克制美学 + 装置感排版 + 机械字体系统**，把"文献工作台"做成一个**有强烈气质、但仍然高度可用的研究装置**——它不是游戏终端，但拥有游戏级的视觉完成度。

用户打开它，第一眼就感受到"这是一个认真对待设计的工具"，而不是"又一个白底黑字的学术页面"。

## B. 设计总方向

### 气质关键词
**工业 · 克制 · 装置 · 机械美学 · 冷峻 · 精密**

不是"温暖学术纸感"，而是"冷静高精度仪器面板"。像一个为严肃研究者设计的专业装置，带有工业设计的冷峻与优雅。

### 核心设计语言：YoRHa DNA 的四个支柱

#### 1. 色彩：黑白灰 + 极低饱和暖调
- 主色调：深灰 `#1a1a1a` / 中灰 `#2d2d2d` / 浅灰 `#f0f0f0` / 纯白 `#ffffff`
- 功能色：极低饱和（去掉鲜艳的绿/红/蓝，改为灰绿 `#5a6b5a` / 灰褐 `#7a6a5a` / 灰红 `#6b4a4a`）
- 强调色：冷白 `#e8e8e8` 与 暗金 `#b8a888`（用于编号与选中态）
- 背景：深色模式为主（`#0a0a0a` 底 + `#1a1a1a` 面板），可选浅色模式（`#f5f5f5` 底 + `#ffffff` 面板）

**策略**：彻底放弃现有的暖米色体系，改为**接近无彩度的灰阶 + 暗金点缀**，营造冷静、专注、工业质感。

#### 2. 字体：Monospace 等宽 + 细 Sans-serif
- 主标题、编号、代码：等宽字体（`JetBrains Mono` / `IBM Plex Mono` / `Consolas` fallback）
- 正文、meta：极细 sans-serif（`Inter` weight 300-400 / `Helvetica Neue` fallback）
- 去掉 Georgia serif——YoRHa 不用衬线，全部用 monospace 与几何 sans

**策略**：论文标题用等宽字体 + 编号前缀（如 `[P-01] Title`），营造"数据条目"感；正文用极细 sans 保证阅读性，整体观感偏向"终端输出 + 精密面板"。

#### 3. 布局：非对称网格 + 浮动标签系统
- 放弃对称三栏，改为**不等宽 + 浮动侧边 module**
- 主区域占据 60%~70% 宽度，侧边模块浮动在右上 / 右下 / 左下，带编号与折叠
- 每个区块带**编号前缀**（`[A] Session Control` / `[B] Paper Index` / `[C] Detail View`），像工业面板的模块标识
- 大量负空间（留白），信息密度低但层级极清晰

#### 4. 材质与细节：扫描线 + 1px 细框 + 微动效
- 背景叠加 CRT 扫描线纹理（`repeating-linear-gradient` 横向细线，opacity 极低）
- 所有边框统一 1px，颜色为半透明灰 `rgba(255,255,255,0.1)` 或 `rgba(0,0,0,0.1)`
- hover 时边框变实、微位移（`translateY(-1px)`）
- 打字机效果用于报告生成、淡入用于论文入栏，动效克制但存在感强

## C. 重构后的布局范式（桌面）

彻底推翻现有三栏等宽 grid，改为**主区 + 浮动模块 + 顶部命令栏**的装置布局。

### 整体结构
```
┌─────────────────────────────────────────────────────────────┐
│ [TOP BAR] 品牌 · 会话选择器 · 状态 · 全局操作              │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  ┌──────────────────────────┐  ┌────────────────┐          │
│  │ [A] SESSION CONTROL      │  │ [D] DETAIL     │          │
│  │ 新建/历史/流程           │  │ 浮动在右上     │          │
│  └──────────────────────────┘  └────────────────┘          │
│                                                               │
│  ┌───────────────────────────────────────────────┐          │
│  │ [B] PAPER INDEX                               │          │
│  │ 主区域：论文列表（编号 + monospace 标题）    │          │
│  │                                                │          │
│  │ [P-01] Retrieval-Augmented Generation...      │          │
│  │ [P-02] Chain-of-Thought Prompting...          │          │
│  │ [P-03] ...                                     │          │
│  └───────────────────────────────────────────────┘          │
│                                                               │
│  ┌────────────────┐                                          │
│  │ [C] FILTER     │                                          │
│  │ 浮动在左下     │                                          │
│  └────────────────┘                                          │
└─────────────────────────────────────────────────────────────┘
```

### 模块说明

#### [TOP BAR] 全局命令栏
- 高度 48px，深色背景 `#1a1a1a`，1px 底边线
- 左侧：品牌标识（`MM` monospace + 小字 `MIAOWEN LITERATURE WORKBENCH`）
- 中部：当前会话选择器（下拉，显示 `[SESSION-01] 当前研究问题`）
- 右侧：全局操作（生成报告 / 导出 BibTeX / 设置）

#### [A] SESSION CONTROL（左上浮动模块）
- 宽度 280px，浮动在左上角，可折叠为细条
- 标题栏 `[A] SESSION CONTROL` 带折叠箭头
- 内容：新建会话表单（textarea + 主按钮）/ 历史会话列表（编号 `[S-01] [S-02]`）/ 流程日志（滚动区）
- 背景半透明 `rgba(26,26,26,0.95)`，细边框

#### [B] PAPER INDEX（主区域）
- 占据中央 60%~70% 宽度，无固定左右边界，两侧留负空间
- 顶部：`[B] PAPER INDEX · 42 RESULTS` + filter 快捷按钮
- 论文行：**编号前缀 monospace 标题**，一行作者 + 小 chips，hover 浮出操作
- 无传统卡片框，每行仅用 1px 底边线分隔，选中时左侧加 2px 暗金竖线
- 滚动区独立，带细 scrollbar

#### [C] FILTER（左下浮动模块）
- 宽度 220px，浮动在左下角，默认折叠为 `[C] FILTER ▸`
- 展开后显示搜索框 + 相关性/状态/年份三个紧凑 select
- 背景同 [A]

#### [D] DETAIL VIEW（右侧浮动模块）
- 宽度 380px，浮动在右上角，可拖动到右下或收起
- 标题栏 `[D] DETAIL VIEW` 可切换 `DETAIL / REPORT` tab
- 内容：选中论文的标题（monospace）/ 摘要（极细 sans）/ 操作 / 引用
- 背景同 [A]，独立滚动

#### [E] QUERY TASKS（可选：右下浮动）
- 检索子任务面板，仅在检索中显示，完成后自动收起
- 编号显示 `[Q-01] Core Search · ✓ Semantic Scholar · ✗ arXiv`

## D. 核心组件重构设计

### D1. 论文行（记忆点）
```
[P-01] ──────────────────────────────────────────────────────
  Retrieval-Augmented Generation for Knowledge-Intensive NLP
  Lewis et al. · 2020 · MUST_READ · Semantic Scholar
  AI: 核心方法论文，提出 RAG 架构...
  ────────────────────────────────────────────────[VIEW][✓]─
```
- **编号前缀** `[P-01]` 暗金 monospace，是唯一视觉锚点
- 标题 monospace 14px，白色/深灰（取决于主题）
- meta 一行，极细 sans 11px，半透明
- AI 理由两行截断，更透明
- hover 整行微亮 + 右侧浮出操作按钮
- 选中时左侧 2px 暗金竖线 `#b8a888`

### D2. 浮动模块通用结构
```
┌─ [A] SESSION CONTROL ───────────────────[ - ][ ✕ ]─┐
│                                                       │
│  内容区                                              │
│                                                       │
└───────────────────────────────────────────────────────┘
```
- 标题栏带模块编号，右侧最小化/关闭图标
- 1px 细边框 `rgba(255,255,255,0.1)`
- 背景半透明深色 `rgba(26,26,26,0.95)` 或浅色 `rgba(255,255,255,0.95)`
- 可拖动、可折叠、折叠后变为细条贴边

### D3. 按钮与交互元素
- 主按钮：深色底 + 白字，1px 白边，monospace 大写文字 `CREATE SESSION`
- 次级按钮：透明底 + 白字，1px 边
- 文本按钮：下划线 hover
- 状态：不用彩色 pill，改为 `[STATUS: READY]` monospace 小标签

### D4. 输入框
- 深色背景 `#2d2d2d`，1px 白边 `rgba(255,255,255,0.2)`
- 内部文字 monospace 或极细 sans
- focus 时边框变实 `rgba(255,255,255,0.6)`

### D5. 报告面板
- 标题 `[REPORT] Research Memo`
- 正文用极细 sans，行高 1.8
- tone 标签改为 `[EVIDENCE]` `[JUDGMENT]` 等 monospace 前缀，无色块
- 生成中打字机效果逐字显示

## E. 移动端策略

YoRHa 风格在窄屏下需简化，但保留核心气质。

### 断点
- `> 1024px`：完整装置布局（主区 + 浮动模块）
- `≤ 1024px`：模块变抽屉，主区占满
- `≤ 768px`：顶部栏收窄，模块全部抽屉化

### 窄屏调整
- 顶部栏保留品牌 + 汉堡菜单，会话选择器收进菜单
- [A] [C] [D] 模块收进左/右抽屉
- [B] 论文列表占满屏宽，编号前缀缩短为 `[01]`
- 浮动模块全部改为覆盖式抽屉，带遮罩

## G. 优先级实施清单

### Phase 1：基础重构（必改，破坏性）
1. `[结构]` 推翻三栏 grid，改为主区 + 浮动模块布局
2. `[样式]` 建立全新 token 体系（深色 + monospace + 灰阶）
3. `[样式]` 论文行改为编号前缀 + monospace 标题
4. `[结构]` 顶部命令栏 + 会话选择器
5. `[样式]` 背景扫描线纹理

### Phase 2：模块化（推荐，提升完成度）
6. `[组件]` 拆分浮动模块为独立组件（SessionControl / FilterPanel / DetailView）
7. `[交互]` 模块拖动、折叠、最小化功能
8. `[动效]` 论文入栏淡入、报告打字机效果
9. `[样式]` 深色/浅色主题切换

### Phase 3：细节打磨（可选）
10. `[交互]` 模块编号系统（`[A] [B] [C]` 可点击跳转）
11. `[样式]` 自定义 scrollbar 样式（极细、半透明）
12. `[动效]` hover 微位移与边框实化
13. `[优化]` 窄屏抽屉动画与手势

## H. 风险、权衡与边界

### 风险
1. **学习曲线**：YoRHa 风格对初次用户有陌生感，需引导或 onboarding。
2. **可读性**：Monospace 标题 + 深色背景对长时间阅读有挑战，需精调字重与对比度，提供浅色模式。
3. **实现复杂度**：浮动模块拖动、折叠、状态持久化比固定三栏复杂，需更多 JS 逻辑。
4. **品牌一致性**：若产品未来需接入学术机构或企业，极致工业风可能显得"过于个性化"。

### 权衡
- **放弃温暖 vs 获得辨识度**：不再是"亲和学术纸感"，而是"冷峻专业装置"，适合追求效率与审美的研究者，不适合保守用户。
- **布局自由 vs 响应式复杂**：浮动模块在桌面很酷，但移动端需全部抽屉化，维护成本高于固定栅格。
- **视觉冲击 vs 信息密度**：YoRHa 靠负空间与层级营造气质，但每屏显示的论文数会少于传统密集列表。

### 明确边界
- **不改变产品定位**：这是文献工作台，不是游戏、不是终端模拟器，YoRHa 风格仅作视觉语言借鉴，核心交互（筛选/精读/报告）保持高效可用。
- **不破坏数据流**：API 契约、SSE 协议、会话/论文/报告数据结构完全不变，重构仅在 UI 层。
- **提供退路**：深色/浅色主题切换，若用户不适应深色可回到浅色；模块布局可提供"经典模式"（三栏固定）作为备选。

## I. 落地路径建议

### 渐进式重构（降低风险）
1. 先在现有三栏基础上换色（灰阶 + monospace），验证可读性与用户反馈
2. 再改布局（主区 + 浮动模块），保留"经典模式"开关
3. 最后加动效与细节打磨

### 原型验证
- 用 Figma / Framer 做高保真原型，测试浮动模块交互与深色模式可读性
- 找 3~5 个目标用户试用，收集"工业风接受度"与"实际使用痛点"

### 技术选型
- 浮动模块可用 `vue-draggable` 或自己实现拖动逻辑
- 主题切换用 CSS 变量 + `localStorage` 持久化
- 扫描线纹理用 CSS `background-image`，性能无忧

---

## 总结

这是一个**大胆、有主张、但仍然可用**的重构方案。它把"文献工作台"从"又一个学术工具"变成"一个让人记住的研究装置"——借鉴 YoRHa 的工业克制美学，但不失去文献筛选的核心效率。

关键取舍：**放弃温暖亲和，换取冷峻辨识度；放弃传统栅格，换取装置感布局；放弃彩色，换取极致克制**。

适合追求审美与效率的研究者，不适合保守学术机构或需要"安全感"的大众用户。若产品定位是"小众但有品味的研究工具"，这套方案值得一试。

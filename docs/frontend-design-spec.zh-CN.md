# 文献妙妙屋 · 前端设计方案（可执行版）

> 本文是基于 `docs/frontend-design-handoff.zh-CN.md` 与当前前端代码（`frontend/src/App.vue`、`style.css`、`services/api.ts`、`main.ts`、`vite.config.ts`）产出的**前端设计说明**。
> 它面向"负责落地的另一个模型或工程师"，只描述设计决策，不含实现代码。
> 执行时优先级：不破坏 API 契约、不破坏独立滚动、不破坏会话/论文/报告数据流、不把产品带偏成聊天或终端 UI。

---

## A. 我对产品的理解

1. 这是一个**科研文献工作台**，核心动作是"输入研究问题 → 多源检索与筛选 → 逐篇精读判断 → 生成研究备忘录 → 本地留存复盘"，不是聊天机器人，也不是搜索引擎结果页。
2. 界面主次非常明确：**中栏论文列表是唯一主舞台**，左栏是轻量索引与启动区，右栏是次级阅读区（详情/报告切换）。
3. 用户的真实停留区是"中栏筛选 + 右栏精读"这条阅读动线，报告是重要但**次级**入口，不能与论文列表争夺视觉中心。
4. 大量"过程性/低优先级信息"（检索子任务、降级提示、数据源状态、流程日志）必须默认收敛，只在需要时展开，绝不能重新膨胀成大卡片。
5. 界面是**动态的**：SSE 事件会实时驱动流程日志、子任务面板、论文入栏、报告拼接，设计必须为"逐步生长的状态"留位置，而不是静态排版。
6. 现有实现已经把视觉往"安静、克制、纸感、工业感"调过一轮（暖米色 + 细线 + serif 标题），本方案是**在既有基线上继续提纯**，不是推翻重做。

## B. 设计总方向

### 气质关键词
安静 · 克制 · 纸感 · 工业感 · 阅读优先。像一张"摊开的学术工作台纸面"，而不是仪表盘或终端。

### 我将遵守的原则
- 保留三栏工作台信息架构，仅做局部提纯与层级重排。
- 保留独立滚动容器思路，中栏列表与右栏阅读永不共用整页滚动。
- 中栏永远是主舞台，右栏永远是次级，报告不越位。
- 低优先级信息一律"缩小、合并、弱化"，默认折叠。
- 视觉语言统一收敛到一套 token，减少一次性特例样式。

### 我刻意不采用的方向（及原因）
- 不做聊天式主界面：会破坏"筛选+精读"的核心动线。
- 不加 hero / landing / marketing 区：这是工具，不是官网。
- 不用霓虹蓝紫科技风、不做侦探终端/档案馆：会滑向"模板化 AI 世界观 UI"，与学术克制气质冲突。
- 不把右栏元信息重做成大块卡片：违反"低优先级压缩"的既定共识。
- 不引入重阴影、大圆角、强渐变：会破坏纸感与工业克制感。

### 页面记忆点（一个就够）
**"编号 + serif 标题的论文行"** 是全局记忆点。左侧大号 serif 序号（`#1 #2 #3`）像论文目录的页码/条目号，配合细线分隔，营造"学术索引册"的独特质感。所有其他区域都为它让路——它是用户视线第一落点，也是产品气质的锚。

## C. 桌面端布局

整体保持三栏 grid：`左 minmax(72px,280px) · 中 1fr · 右 minmax(72px,420px)`。收起态统一收缩到 `72px`。中栏永远最宽、最亮、信息密度最高。

### 三栏视觉权重分配
- **中栏 = 100% 注意力**：背景最亮（surface-strong），内容密度最高，是唯一"内容滚动区"。
- **左栏 ≈ 20% 权重**：背景比中栏略深半档，字号更小，作为"索引 + 启动"薄层。
- **右栏 ≈ 35% 权重**：背景与左栏同档，是"阅读抽屉"，永远不与中栏抢亮度。

关键手法：**用背景明度差建立主次**，而不是用边框粗细或色彩。中栏最亮，两侧略沉，视线自然被拉向中间。

### 左栏：更像索引与入口
- 品牌区压到**单行 lockup**：`MM` 方块 + "文献妙妙屋" 一行标题，副描述降为一行 11px eyebrow，收起时只留方块居中。
- 新建会话表单是左栏**唯一的"主行动"**：textarea + 一个深色主按钮，其余全部弱化。表单外框去掉，改为顶部一条细分隔线，减少"卡片套卡片"。
- 历史会话 / 流程记录用**分组标题（小 caps eyebrow）+ 无外框列表**呈现，让左栏像一叠便签，不像堆叠卡片。
- 左栏整体降低对比：hover 才浮出细边，选中态用左侧 2px 竖线 + 背景微亮，而非整框高亮。

### 中栏顶部固定区：继续压缩
目标是**把垂直空间还给论文列表**。当前 header + banner + query-board + filter-bar 四段固定区偏高。
- 会话头部压成**两行**：第 1 行 = 状态 pill + serif 标题（单行省略）；第 2 行 = 一条 12px meta（`n/N 已纳入 · k 份报告 · 数据源摘要`）。副标题的数据源明细移入可 hover 的 tooltip 或折叠，不占固定行。
- 主操作按钮（生成报告 / 打开报告）右对齐到标题行，做成**图标+短文字**的紧凑按钮，报告类按钮视觉权重低于"生成报告"。
- 降级提示、检索子任务保持**默认折叠**，收起时只占一行 summary（`降级提示 · 2 项` / `检索子任务 · 5 组`）。两者可并入同一行"过程信息条"，进一步省高度。
- filter-bar 从"四个带 label 的输入"压成**一行紧凑工具条**：搜索框占主，相关性/状态/年份三个 select 做成短下拉，label 内嵌为 placeholder 或前缀小字，去掉每个 select 上方的独立 label 行。

固定区总目标：从当前约 4~5 段压到"标题两行 + 一条过程信息条 + 一条 filter 条"，给列表多让出至少一屏 15%~20% 高度。

### 中栏论文列表：更清晰的信息层级
论文行是产品核心，建立稳定的三级信息节奏：
- **第一级（扫读）**：serif 大号 rank 序号 + 标题 + 年份。
- **第二级（判断）**：一行作者 + 一行 chips（相关性标签在最前，来源/tags 在后，低饱和）。
- **第三级（辅助）**：AI 判断理由 2 行截断，颜色降为 muted。
- 右侧操作区（查看/纳入）改为**hover 才完全显现**，静止时只留一个已纳入的实心小点状态，减少每行的视觉噪声。
- 行与行之间用**细线分隔 + 适度留白**，而不是每行独立描边卡片，强化"目录/索引册"观感。选中行用左侧竖线 + 背景微亮。

### 右栏 detail / report：更像阅读面板
- detail 与 report 用**顶部一组极简 tab 切换**（下划线式，不是实心按钮组），弱化"控制台"感。
- **detail 首屏 = 阅读**：hero（serif 标题 + 一行作者 + 一行 meta + 评分 pill）→ 摘要（大行高 reading-copy，作为视觉主体）→ AI 判断（虚线分隔的附注）。打开论文 / BibTeX / 状态操作收拢为标题下方**一行紧凑操作条**，不做成大按钮阵列。
- 引用块、检索命中、派生入口按"越往下越次级"排列，派生表单放最底部，存在感最弱。
- **report = 研究备忘录侧栏**：hero 更轻（一行 eyebrow + serif 标题 + 已确认数 badge），TOC 做成一行 wrap 的细 chip，正文按 tone（证据/判断/推测/行动/说明）用**左侧一条 tone 色线 + 小标签**区分，而不是大色块卡片。report 的整体亮度与密度都低于中栏列表，确保不越位。
- 两个面板各自独立滚动（保留 `panel-scroll`），切换 tab 与选中论文时滚动回顶（保留现有行为）。

## D. 移动端 / 平板 / 窄屏策略

沿用现有断点并明确各档优先级。**任何断点下，中栏论文列表都是最高优先级。**

### 断点与行为
- **> 1180px（桌面三栏）**：完整三栏，右栏常驻。
- **≤ 1180px（平板 / 小桌面）**：右栏改为**覆盖式抽屉**（fixed，右侧滑入），收起时贴边只露 72px 把手。左栏保留但可收窄。中栏拿到主要宽度。
- **≤ 960px（手机 / 竖屏）**：整体堆叠为单列纵向流，顺序 = 会话头部 → filter → 论文列表 → 右栏内容按需抽屉/切换。左栏折叠为顶部一条"会话入口栏"。

### 各区收纳方式
- **左栏**：窄屏收成顶部一条 bar（品牌 + "新建/历史"入口按钮），点开为覆盖式面板，不占常驻纵向空间。
- **右栏**：始终抽屉化，从右侧滑入，带遮罩；detail/report tab 在抽屉内切换。选中论文自动打开抽屉到 detail。
- **filter-bar**：窄屏折叠为"搜索框 + 一个筛选按钮"，点开展开其余下拉，避免占三四行。

### 降级 / 隐藏优先级
- 优先隐藏：检索子任务面板（默认折叠已足够）、数据源明细、流程日志（收进左栏抽屉）。
- 保留：论文标题、年份、相关性标签、纳入操作——这几项是移动端筛选的最小可用集。
- 论文行在窄屏下从三列 grid 塌成单列：序号缩小内联到标题前，操作区横排到底部。

## E. 组件级建议

逐个组件给出"现状问题 → 设计建议"。

### E1. 左栏品牌区（brand-lockup）
- 现状：方块 + 三行文字（eyebrow / 标题 / 描述），略占高。
- 建议：压成两行——`MM` 方块 + serif 标题一行，eyebrow 缩到方块右侧小字或省略描述。收起态只留方块居中。品牌区高度减半，把空间让给新建会话。

### E2. 新建会话区（topic-form）
- 现状：带外框卡片，主按钮 + 取消按钮。
- 建议：去外框，改为顶部区块（一条细分隔线 + 小标题 "新建调研"）。textarea 保持 5 行，主按钮全宽深色，检索中时按钮内联 loading 文案，"取消"降为按钮右侧的小 text-btn。这是左栏唯一强调项。

### E3. 历史会话项（session-item）
- 现状：网格 + 状态 pill + meta + 删除 ×。
- 建议：无外框列表行，hover 浮出细边与删除按钮（静止时删除 × 极淡）。选中态用左侧 2px 竖线。状态用**小圆点 + 文字**替代 pill，降低色块密度。meta（`n/N · 时间`）压成一行 11px muted。

### E4. 流程日志区（progress-list）
- 现状：常驻列表，可清空。
- 建议：默认只显示最近 3~5 条，其余折叠为"展开全部"。每条前的方点保留（工业感细节）。整体作为左栏最底、最弱的一块，字号 12px muted。窄屏收进抽屉。

### E5. 中栏会话头部（stage-header）
- 现状：标题 + 两行 meta + 两个操作按钮，占高偏多。
- 建议：压两行（见 C）。标题 serif 单行省略；meta 一行；操作按钮右对齐紧凑化。"生成研究报告"为唯一主按钮，"打开报告"降为次级 ghost。

### E6. query board / task board（检索子任务）
- 现状：可折叠，展开是双列卡片网格，信息很密。
- 建议：**默认折叠**保持。收起态只占一行 summary（`检索子任务 · N 组`）。展开时卡片弱化外框，改为细线分隔的条目；每个 variant 的"成功/跳过/失败源"那行长文本压成 hover 展开或图标计数（`✓3 ⊘1 ✕0`），避免长串灰字撑高。这是"过程性信息"，视觉权重必须低于论文列表。

### E7. filter bar
- 现状：四个带独立 label 的输入，占一整段 grid。
- 建议：压成一行工具条。搜索框带放大镜前缀占主宽；相关性/状态/年份三个 select 紧凑排右侧，label 内嵌为前缀小字或 placeholder。窄屏折叠为"搜索 + 筛选按钮"。

### E8. paper row（核心）
- 现状：`64px 序号 | 主体 | 92px 操作` 三列。
- 建议：保持三列骨架但强化层级——序号更 serif、更大、accent 色；标题为行内视觉主角；chips 低饱和且相关性标签置首；AI 理由 muted 两行截断；操作区 hover 显现，静止时仅留"已纳入"实心点。行间细线分隔取代独立卡片描边。这是记忆点，投入最多打磨。

### E9. detail hero
- 现状：eyebrow + 评分 pill + serif 标题 + 作者 + 一行 meta（年份/venue/引用/来源）。
- 建议：结构对，保持。微调：评分 pill 右上角，meta 用 `·` 分隔的一行小字（已实现），确保标题是绝对视觉主体。hero 下方不放大按钮，操作条另起一行紧凑排列。

### E10. abstract block（摘要）
- 现状：摘要 + AI 判断附注在同一块。
- 建议：摘要是**右栏阅读主体**，给最大行高（1.75+）与最舒展留白，字号略大于其他正文。AI 判断保持虚线分隔的次级附注，muted 色，标签 "AI 判断" 小字加粗。两者主次要拉开。

### E11. citation / metadata block（引用/元信息）
- 现状：引用框 + 复制 + BibTeX，检索命中列表。
- 建议：引用块保持紧凑单框；BibTeX 作为次级链接保留（尊重"BibTeX > Markdown"共识）。检索命中列表默认可折叠（`检索命中 · N 条`），展开才显示 query 明细。元信息**不做成大卡片墙**，全部保持一行 meta 或折叠。

### E12. report block（研究报告）
- 现状：hero + TOC chips + 按 tone 分类的正文条目。
- 建议：hero 更轻；TOC 一行 wrap 细 chip；正文用**左侧 tone 色线 + 小 tone 标签**区分证据/判断/推测/行动/说明，不用大色块。整体亮度/密度低于中栏。保留 report 容器内独立滚动与 TOC 跳转（不滚整页）。生成中流式拼接文本时保持 `report-draft` 等宽降级视图。

### E13. 折叠控件（rail-toggle / query toggle）
- 现状：竖向把手（三横线点 + 箭头），query board 用文字"展开/收起"。
- 建议：左右栏把手保持图形化（符合"不要大字按钮"共识），统一为**贴边细长把手 + 方向箭头**，hover 才加深。query board / 命中 / 降级等区内折叠统一用"标题右侧箭头图标 + summary 计数"，去掉"展开/收起"文字，视觉更一致克制。

### E14. 错误 / 降级提示（banner）
- 现状：error 红条 + degradation 折叠 details。
- 建议：error 保持醒目但克制（细红边 + 红字，不做实心红块）。降级提示默认折叠为一行，可与检索子任务合并进同一条"过程信息条"，进一步省中栏顶部高度。

### E15. 空状态 / loading 状态
- 现状：虚线框 empty-state；创建/生成用按钮文案切换。
- 建议：空状态保持虚线框 + 两行文案的克制风格，文案引导下一步（"先创建会话"/"选择一篇论文"）。loading 用**骨架行**（列表出 3~4 条灰底占位行）替代空白，配合 SSE 逐条入栏形成"生长感"。流程按钮 loading 内联文字 + 细进度点即可，不加大 spinner。

## F. Design Tokens

在现有 `.workspace` CSS 变量基础上收敛与补齐。**沿用当前色值**，仅补充语义与状态 token。

### 颜色 token（保留现有基线）
```
--bg:            #e8dfcf   /* 页面纸感底，带细横纹 */
--surface:       rgba(243,236,225,0.96)  /* 左右栏薄层，比中栏略沉 */
--surface-strong:rgba(250,246,238,0.98)  /* 中栏 / 内容卡，最亮 */
--line:          #b2a388   /* 主边框、把手线 */
--line-soft:     rgba(90,74,52,0.18)      /* 分隔线、弱边框 */
--text:          #201a14   /* 正文 */
--muted:         #6f6557   /* 次级文字、meta */
--accent:        #4f4030   /* 序号、链接、强调线 */
--accent-strong: #20160f   /* 主按钮底、选中底 */
--success:       #35573b   /* 核心/证据 */
--warning:       #916a28   /* 前沿/行动/降级 */
--danger:        #923939   /* 错误/删除/排除 */
--speculation:   #6c4a87   /* 推测/旁支（现有隐性色，提为 token）*/
```
配色策略：**单一暖棕色系 + 三个功能色（绿/褐/红）+ 一个低饱和紫**。功能色只用于文字与细线，绝不铺大色块。

### 字号层级
```
display   24px  serif   会话/详情/品牌大标题
title     18-22px serif  报告标题、对话框标题
h-section 14px  sans    区块小标题
body      13-14px sans   正文、摘要（摘要可到 15px）
meta      12px  sans     作者、meta、chips
micro     11px  sans     eyebrow、时间、计数
rank      22px  serif    论文序号（记忆点）
```

### 字重
```
标题 serif        400（依赖 serif 字形，不加粗）
区块标题/强调      600
按钮/tab active    600
正文/meta          400
eyebrow            500 + letter-spacing 0.06em + uppercase
```

### 行高
```
标题        1.15
区块标题    1.2
正文        1.5
摘要/报告   1.75  （阅读区专用大行高）
meta        1.4
```

### spacing scale（4 基准）
```
xs 4 · sm 8 · md 12 · lg 16 · xl 22 · 2xl 32
```
规则：区块内间距用 8/12，区块间用 16，栏内边距 18，中栏外边距 22。**统一到这套刻度**，消除现有 6/10/14 等零散值。

### border thickness
```
默认边框        1px
选中/激活强调    左侧 2px 竖线（不加粗整框）
把手线          1px
```
策略：**只用 1px 细线**，层级靠明度与虚实（实线 vs 虚线 `line-soft`）区分，不靠粗细。

### radius strategy
```
--radius: 6px  统一用于按钮、输入、chip、卡片
pill 类（状态/标签）：仍用 6px 直角偏方，不做全圆角
```
策略：小圆角克制统一，强化工业纸感，**不出现 12px+ 大圆角**。

### shadow strategy
```
常态：无阴影（靠边框与明度分层）
浮层：对话框 / 移动端右栏抽屉 才用 ——
  抽屉  -10px 0 30px rgba(25,19,14,0.14)
  对话框 0 8px 30px rgba(24,18,14,0.22)
```
策略：**扁平优先**，阴影仅用于真正脱离纸面的浮层，绝不给普通卡片加投影。

### 交互状态 token
```
hover:    背景微亮 +2% 或浮出 line-soft 细边
active:   accent-strong 底 + 浅色字（主按钮/tab/状态选中）
selected: 左侧 2px accent 竖线 + 背景 rgba(255,245,229,0.9)
focus:    outline 2px rgba(79,64,48,0.28) + offset 2px（保留现有）
disabled: opacity 0.5 + cursor not-allowed，不变色
excluded: opacity 0.58（保留现有）
```

## H. 风险与假设

### 假设
1. **假设后端 SSE 事件与字段不变**：本方案所有"折叠/计数/tone"展示都基于现有 `queries / source_statuses / degradation_notices / report tone 前缀` 结构，未要求任何新字段。
2. **假设报告 markdown 结构稳定**：`parseReportMarkdown` 依赖 `#`/`##`/`- `/`证据：` 等前缀，report 的 tone 呈现建立在此约定上。若后端改文案前缀，tone 分类会失效——建议保持前缀约定。
3. **假设单文件组件可继续承载**：现状全部在 `App.vue`，本方案默认不强制拆分；G14 的拆分是可选项，不是前提。
4. **假设无需引入组件库/字体依赖**：serif 沿用 Georgia 系统字体栈，等宽沿用 Cascadia/Consolas，不新增 web font，避免构建级改造。

### 风险与冲突点
1. **顶部压缩 vs 信息完整**：把降级提示、数据源明细收进折叠/hover 后，"降级发生"的可感知度下降。缓解：过程信息条在有降级时用 warning 色点提示，不完全隐形。
2. **hover 才显操作 vs 触屏**：论文行"hover 显操作"在触屏无 hover。缓解：窄屏/触屏断点下操作区常驻横排（已在 D 节说明）。
3. **明度差主次 vs 对比度**：左右栏压暗不能过度，否则次级文字对比不足。缓解：muted 文字维持在 `#6f6557` 及以上对比，正文始终 `--text`。
4. **骨架行 vs 现有 SSE 逐条渲染**：若后端在 `screening_done` 前不发增量论文，骨架行只能是固定数量占位，无法反映真实进度。缓解：骨架仅作视觉占位，真实进度仍由流程日志承担。
5. **serif 序号跨字体一致性**：Georgia 在不同系统字形略有差异，可能影响记忆点观感。缓解：接受系统字体差异，或后续按需引入单一 serif web font（属可选、非必需）。

### 明确不在本方案范围
- 不新增后端功能、不改 API 契约、不改 SSE 协议。
- 不把产品改成聊天 / landing / dashboard / 终端 UI。
- 不把中栏列表与右栏阅读重新绑成整页滚动。
- 不要求更换框架、引入 router 或状态管理库作为落地前提。
